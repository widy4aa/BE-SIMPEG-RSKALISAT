8) API LOGIN (NIK + PASSWORD + JWT)
-----------------------------------
Endpoint:
- POST /api/login

Arsitektur yang digunakan:
- Controller: app/Http/Controllers/Api/AuthController.php
- Service: app/Services/Auth/AuthService.php
- Repository: app/Repositories/Auth/AuthRepository.php
- Form Request (validasi): app/Http/Requests/Auth/LoginRequest.php
- JWT helper service: app/Services/Security/JwtService.php
- JWT config: config/jwt.php

Route:
- routes/api.php
	- Route::post('/login', [AuthController::class, 'login']);

Request body login:
{
	"nik": "3174010101010001",
	"password": "password"
}

Validasi input:
- nik: required|string|max:30
- password: required|string|min:6

Response sukses (HTTP 200):
{
	"success": true,
	"message": "Login berhasil.",
	"data": {
		"token_type": "Bearer",
		"access_token": "<jwt_token>",
		"expires_in": 43200,
		"user": {
			"id": 1,
			"nik": "3174010101010001",
			"role": "pegawai",
			"nama": "Budi Santoso"
		}
	}
}

Response error yang ditangani:

1. Validasi gagal (HTTP 422)
{
	"success": false,
	"message": "Validasi gagal.",
	"errors": {
		"nik": ["NIK wajib diisi."],
		"password": ["Password wajib diisi."]
	}
}

2. NIK/password salah (HTTP 401)
{
	"success": false,
	"message": "NIK atau password tidak valid."
}

3. Akun tidak aktif (HTTP 403)
{
	"success": false,
	"message": "Akun tidak aktif. Silakan hubungi admin."
}

Catatan keamanan JWT:
- Token ditandatangani algoritma HS256.
- Secret diambil dari JWT_SECRET (fallback ke APP_KEY jika JWT_SECRET belum diisi).
- Durasi token diatur lewat JWT_TTL (default 43200 detik / 12 jam).
